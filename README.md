## 📖 О проекте

**Vernam-ML-KEM** — это полноценная демонстрационная криптосистема, сочетающая **постквантовую стойкость** алгоритма ML-KEM (стандарт NIST 2024, Kyber-768) с **абсолютной информационно-теоретической стойкостью** классического шифра Вернама (One-Time Pad).

Проект реализует концепцию **гибридного шифрования**:
- ML-KEM обеспечивает безопасную передачу общего секрета (shared secret) даже в мире квантовых компьютеров.
- HKDF (RFC 5869) растягивает секрет в OTP-ключ нужной длины.
- Шифр Вернама (XOR) обеспечивает идеальную секретность при однократном использовании ключа.
- HMAC-SHA256 добавляет аутентификацию и защиту от подмены данных.

### Основные возможности
- **4 уровня шифрования** (L1–L4):
  - **L1** — Предпросмотр (только первые 100 символов)
  - **L2** — Verbose (полный лог всех переменных + побайтовый XOR)
  - **L3** — Reversible (два файла: оригинал + tampered с обнаружением изменений)
  - **L4** — **ML-KEM FULL** (постквантовый ★ IDEAL)
- Два интерфейса:
  - **Графический (Tkinter)** — `gui_app.py`
  - **Веб-интерфейс** — симуляция классической модели Alice–Bob–Eva (`/alice`, `/bob`, `/eva`)
- Симуляция реального канала связи с TTL 20 секунд
- Полная визуализация всех шагов (keygen → encaps → HKDF → XOR → MAC)
- Побайтовый лог шифрования
- Обнаружение tampering (L3)
- Поддержка текста и произвольных файлов

## ✨ Особенности

- **Постквантовая стойкость** — ML-KEM (Module-LWE) устойчив к атакам Шора и Гровера.
- **Информационно-теоретическая стойкость** — OTP при правильном использовании.
- **HKDF + HMAC-SHA256** — современные стандарты KDF и MAC.
- **Чистый Python** — без внешних зависимостей (кроме Flask/Tkinter).
- **Полная симуляция** — ML-KEM реализован как детерминистический симулятор (Kyber-768 параметры).
- **Красивый UI** — тёмная тема, анимации, Orbitron + Space Mono.
- **Кросс-платформенность** — Windows / Linux / macOS.
